This page lists the YAML keys validated inside `ByobConfig.from_yaml` in `runtime/config.py`.
Optional keys show their defaults in the dataclass or in the `get` calls inside `from_yaml`.

## Experiment layout

| Key | Notes |
| --- | --- |
| <code>expt_name</code> | Required string; builds <code>output_dir/expt_name/</code>. |
| <code>output_dir</code> | Required writable directory. |
| <code>input_dir</code> | Required path containing each <code>target_source_mapping</code> corpus. |
| <code>random_seed</code> | Optional; seeds Python and NumPy when present. |
| <code>ndd_batch_size</code> | Optional positive int; defaults to <code>1000</code> in the dataclass but sample configs use smaller values. |

## Dataset selection

| Key | Notes |
| --- | --- |
| <code>hf_dataset</code> | Must appear in <code>ALLOWED_HF_DATASETS</code>. |
| <code>subset</code> | Optional; defaults per [Supported Hugging Face Benchmarks](/benchmarks). |
| <code>split</code> | Optional; defaults to <code>"test"</code>. |
| <code>language</code> | Required string (for example <code>en-US</code>). |
| <code>metadata_file</code> | Optional CSV path enabling tag-aware sampling. |

## Subjects and sampling

| Key | Notes |
| --- | --- |
| <code>source_subjects</code> | Required non-empty list of benchmark subject strings. |
| <code>target_source_mapping</code> | Required mapping; see [Prepare Your Own Domain Data](/../how-to/prepare-data). |
| <code>few_shot_samples_per_query</code> | Required positive int. |
| <code>queries_per_target_subject_document</code> | Required positive int. |
| <code>num_questions_per_query</code> | Required positive int. |

## Prompts and models

| Key | Notes |
| --- | --- |
| <code>prompt_config</code> | <code>null</code> loads packaged defaults; a string path loads YAML with all required stages. |
| <code>generation_model_config</code> | Required mapping for Data Designer calls. |
| <code>judge_model_config</code> | Required mapping. |
| <code>do_distractor_expansion</code> | Required bool. |
| <code>distractor_expansion_model_config</code> | Required when expansion is true. |
| <code>distractor_validity_model_config</code> | Required mapping. |
| <code>filtering_model_configs</code> | Required dict with <code>hallucination</code> and <code>easiness</code> lists. |
| <code>easiness_threshold</code> | Required float in <code>[0, 1]</code>. |
| <code>hallucination_threshold</code> | Required float in <code>[0, 1]</code>. |
| <code>remove_hallucinated</code> | Optional bool; defaults to <code>True</code>. |
| <code>remove_easy</code> | Optional bool; defaults to <code>False</code>. |

## Optional quality stages

| Key | Notes |
| --- | --- |
| <code>semantic_deduplication_config</code> | Dict with <code>enabled</code>, embedding model id, clustering parameters, and <code>remove_duplicates</code>. |
| <code>semantic_outlier_detection_config</code> | Dict with <code>enabled</code>, embedding model id, and neighbour thresholds. |
| <code>chunking_config</code> | Dict; supports <code>window_size</code> (<code>null</code> keeps whole documents). |
| <code>do_coverage_check</code> | Bool; defaults to <code>False</code> when omitted. |
| <code>coverage_check_config</code> | Dict with <code>window_size</code> and <code>model_identifier</code>. |

## Curator mount

Sample configs include a `run.env.mounts` entry that uses `$\{auto_mount:...\}` to place NeMo Curator on `/opt/Curator`.
Remote profiles must preserve the same mount contract your environment expects.

## Stage dispatch for `nemotron steps run`

The generic steps CLI accepts `family`, `stage`, and `skip_until` as dotlist overrides, and you can also place those keys directly in YAML.
`family` defaults to `mcq` when omitted.
