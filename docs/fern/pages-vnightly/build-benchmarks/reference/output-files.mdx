All paths below are relative to `output_dir` from your YAML and the string `expt_name`.

## Prepare

| File | Description |
| --- | --- |
| <code>seed.parquet</code> | Few-shot rows plus domain chunks. |

## Generate

The final generation creates these files:

| File | Description |
| --- | --- |
| <code>benchmark_raw.parquet</code> | Snapshot immediately before column renaming for the final schema. |
| <code>benchmark.parquet</code> | Final MCQ schema; column meanings and a sample row are in the next section. |

The following intermediate files that are created in the `stage_cache` directory:

| File | Stage |
| --- | --- |
| <code>generated_questions.parquet</code> | GENERATION |
| <code>judged_questions.parquet</code> | JUDGEMENT |
| <code>semantic_deduplicated_questions.parquet</code> | SEMANTIC_DEDUPLICATION |
| <code>expanded_distractors.parquet</code> | DISTRACTOR_EXPANSION (only when <code>do_distractor_expansion</code> is true) |
| <code>coverage_check.parquet</code> | COVERAGE_CHECK (only when <code>do_coverage_check</code> is true) |
| <code>valid_distractors.parquet</code> | DISTRACTOR_VALIDITY_CHECK |
| <code>semantic_outlier_detection.parquet</code> | SEMANTIC_OUTLIER_DETECTION |
| <code>filtered_questions.parquet</code> | HALLUCINATION_EASINESS_DETECTION |

## Final MCQ Columns

Generation and translation both export the same eight columns on the final `benchmark.parquet` file.

| Column | Meaning |
| --- | --- |
| <code>question_id</code> | Stable identifier for the row, taken from the internal <code>id_question</code> field before export. |
| <code>question</code> | Stem text for the multiple-choice item. After translation this is the target-language text. |
| <code>options</code> | Ordered list of choice strings. The list order matches the letter labels implied by <code>answer_index</code>. |
| <code>answer_index</code> | Zero-based index into <code>options</code> for the correct choice. |
| <code>answer</code> | Letter label for the correct choice, derived from <code>answer_index</code> (<code>0</code> → <code>A</code>, <code>1</code> → <code>B</code>, and so on). |
| <code>cot_content</code> | Reserved for chain-of-thought text. The current pipeline sets this column to the literal <code>-</code> for every row on export. |
| <code>src</code> | Reserved for a source document marker. The current pipeline sets this column to the literal <code>-</code> for every row on export. |
| <code>category</code> | Target key from <code>target_source_mapping</code> in your generation configuration, in other words the domain bucket name for the row, not the Hugging Face few-shot subject name. |

### Sample Row

Values below are illustrative; your identifiers and wording will differ.

```json
{
  "question_id": "mcq-00042",
  "question": "If x^2 = 9, which value can x take?",
  "options": ["-3 only", "-3 or 3", "3 only", "9"],
  "answer_index": 1,
  "answer": "B",
  "cot_content": "-",
  "src": "-",
  "category": "maths"
}
```

## Translate

The translate stage creates the following final output files:

| File | Description |
| --- | --- |
| <code>benchmark_raw.parquet</code> | Intermediate snapshot prior to optional quality filtering. |
| <code>benchmark.parquet</code> | Final translated MCQ after fields are renamed back to <code>question</code>, <code>options</code>, <code>answer_index</code>, and <code>answer</code>. Column semantics match generation; see [Final MCQ Columns](#final-mcq-columns). |

The following intermediate files are created in the `staged_cache` directory.

| File | Stage |
| --- | --- |
| <code>translated_questions.parquet</code> | TRANSLATION |
| <code>backtranslated_questions.parquet</code> | BACKTRANSLATION |
| <code>quality_metrics.parquet</code> | QUALITY_METRICS |

Intermediate translation Parquet files can include additional columns such as `question_translated`, `options_translated`, backtranslation fields, and metric scores.
