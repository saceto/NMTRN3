This page describes the YAML configuration file that you provide for the translate stage: the keys you set or override, how backtranslation quality scores show up in Parquet output columns, and which rows are omitted from the final `benchmark.parquet` when quality filtering is enabled.

## Required Keys

| Key | Notes |
| --- | --- |
| <code>expt_name</code> | Directory name under <code>output_dir</code> for caches and final artifacts. |
| <code>dataset_path</code> | Existing <code>benchmark.parquet</code> from a generation run. |
| <code>output_dir</code> | Parent directory for <code>expt_name</code>. |
| <code>source_language</code> | BCP-47 style tag (for example <code>en-US</code>). |
| <code>target_language</code> | Target locale tag (for example <code>hi-IN</code>). |
| <code>translation_model_config</code> | Dictionary with <code>backend_type</code>, <code>params</code>, and optional <code>stage</code> and <code>segment_stage</code>. |
| <code>backtranslation_quality_metrics</code> | Non-empty list; each element is a dictionary with <code>type</code> and <code>threshold</code>. |

## Quality Metrics

Each `type` must be `sacrebleu`, `chrf`, or `ter`.
Each `threshold` must be nonnegative.

NeMo Curator writes one numeric score column and one boolean pass column per configured metric, for example `score_chrf` and `score_chrf_passed`.
The column `is_quality_metric_passed` is true on a row when every per-metric pass column is true for that row.

Each score compares the original benchmark text with the round-trip backtranslation from the target locale, using sentence-level APIs from the *sacrebleu* library.

| <code>type</code> | Measures | Scale and direction | How to interpret scores | Row passes when |
| --- | --- | --- | --- | --- |
| <code>sacrebleu</code> | Sentence bilingual evaluation understudy (BLEU) | 0 through 100 after *sacrebleu* tokenization; higher values track closer matches to the reference. | High scores mean the backtranslation preserved wording and order; scores near zero mean little *n*-gram overlap. | <code>score_sacrebleu</code> ≥ <code>threshold</code> |
| <code>chrf</code> | Sentence character n-gram F-score (chrF) | 0 through 100 in typical sentence outputs; higher values mean closer character-level match. | High scores track spelling and phrasing fidelity; low scores mean the backtranslation diverged on the surface string. | <code>score_chrf</code> ≥ <code>threshold</code> |
| <code>ter</code> | Sentence translation error rate (TER) | Zero means no edits; larger values report more insertions, deletions, or substitutions relative to the reference. | Values close to zero mean the backtranslation needed minimal editing to match the original; large values signal heavy rewrites or mismatch. | <code>score_ter</code> ≤ <code>threshold</code> |

Inspect `stage_cache/quality_metrics.parquet` under your experiment directory to pick thresholds from the score spread you see in data.

## Optional Keys

| Key | Default | Notes |
| --- | --- | --- |
| <code>remove_low_quality</code> | <code>True</code> unless YAML overrides it | When true, the pipeline omits rows where <code>is_quality_metric_passed</code> is false before export. |

## FAITH evaluation

`translation_model_config.stage.enable_faith_eval` must not be true.
The benchmark translation relies on backtranslation metrics instead of FAITH filtering
